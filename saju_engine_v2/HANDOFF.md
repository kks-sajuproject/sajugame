## 0-E. 5단계 심화 (2026-06-22~) — 사용자 요청 [1]→[4] 순차

### [1] 격국 고도화 + 통계검증 ✅
- `geukguk.py` **`classify()`**: 화기격 → 종격 → 정격(월령) 순 종합판정. 종격(從財·從殺·從兒·從旺·從強)은 **score ≤ -4.5 & 부조세력 ≤ 1 & 무근**(`rooted()`: 지지 지장간에 비겁/인성 전무)일 때만 眞從. 假從 오판 방지(증국번 score-4.7이지만 寅에 통근 → 칠살격 유지). 월령이 비겁(건록/양인)이면 `alt_geuk()`로 타주 격취.
- **`johu()`**: 궁통보감 요지 월령별 조후용신(亥子월→火, 巳午월→水 등).
- **命例 통계검증**(결과태그 ↔ classify): 大貴(n=219)는 강약 고름(신강76·중화64·신약79) → **귀천은 강약 무관, 격국 청탁이 결정**(명리 통설 데이터 확인). 壽=중화 우세(8중7), 夭=신약 우세, 貧=신강(비겁과다) 우세. → 강약 모델은 수명·빈부엔 유효, 귀천 판정엔 부적합(의도된 한계).
- `measure_geukguk.py` 종격 포함 재측정: 계열 50.6%·엄밀 39.2%. (월령 외 격취 고급케이스가 상한)
- `analyze(간지4)` 반환에 classify·yongsin·johu 통합.

### [2] 웹앱 해석문 + 고전 근거 ✅
- JS에 `classify()`(화기격·종격·월령외)·`johu()`·`powerProfile()`·`rooted()` 동기화 → Python과 격국·강약·조후 **완전 일치 검증**(손중산 정격 상관격 신약-3.2 조후火, 증국번 칠살격 등).
- **종합 해석문 자동생성**(`interpret()`): 일간·강약·격국·용신·조후·신살·오행균형을 한국어 문장으로. "일간이 辛(금)이며 신약… 도와주는 운을 기뻐… 상관격… 조후 火" 식.
- **고전 근거 임베드**: `build_refs.py` → `refs_table.json`(격국·강약·신살·용신 36키워드 69구절, saju_full.db에서 발췌, 27KB). 단일 HTML이라 서버 없이 분석결과 격국/신살에 맞춰 표시. ⚠일부 근거가 키워드 단순매칭이라 성별·맥락이 어긋날 수 있음(후속 정교화 여지).
- `saju_web.html` 84.9KB. `build_webapp.py`에 REFS 주입.

### [3] 운세: 세운·대운 길흉 ✅
- `fortune()`/`fortuneCard()`: 억부 용신 오행 기준으로 향후 10년 세운 + 대운 길흉 판정(용신 부합 ○길 / 기신 ●주의 / △평). 십신 → 분야(FIELD: 재물·관운·학업·도전 등) 매핑해 "가까운 길운에 ○○ 방면 순조" 요약.
- 기존 대운 카드를 길흉 통합 카드로 교체. saju_web.html 87.7KB.

## 3. 번역 작업 방식 (반드시 이 방식 유지)

### 절차
1. 미번역 원문 추출 → 2. 한국어 완역 → 3. 병합 스크립트로 JSON 갱신 → 4. 검증 → 5. 뷰어 재생성

### 새 병합 스크립트 템플릿 (예: 다음 dts 배치)
기존 `build_tr_dts2.py`에 `NEW.update({...})` 블록을 **`# === MARKER ===` 위에** 계속 추가하면 됨.
형식: `id: {"t":"절제목(한자 병기)", "b":"한국어 본문"}`

### 번역 스타일·관례
- 문체: 고전 직역체("~한다/~이다"), 핵심 명리용어는 한자 병기(예: 식신(食神)).
- **적천수(천미본) 구조**: 본문(經) → `▷ 원주(原注): …` → `▷ 임씨(任氏)가 말하였다. …`
- **命例(사주 예시)**: PDF 추출 과정에서 사주 기둥(예: `癸酉 / 己亥壬申丙子`)이 본문과 뒤섞여 있음. → **풀이(논단) 위주로 옮기고 판독 곤란한 간지 나열은 `〔명례 — …〕`로 요약 표시**.
- 한 절이 다음 절로 이어지면 끝에 `(다음id로 이어진다)`, 앞에서 이어지면 `〔앞id에서 이어진다〕` 표기.
- 운문(賦·歌·訣)은 뜻을 살린 산문 완역, 편집자 보충은 `〔 〕`.
- yhzp는 원문텍스트본↔PDF본 2판본이 동일내용이라 `id`와 `id+136`에 같은 번역 복제(yhzp 전용). dts는 단일 판본이라 복제 없음.

### 검증 항목
- 전체 절수 = JSON 키 수(누락 0), 빈/과단(<20자) 항목 0, 미번역 한문 덩어리(12자+ 연속) 0.
